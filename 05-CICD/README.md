# Module 5 – CI/CD Tools (GitHub Actions, GitLab CI/CD, Jenkins)

CI/CD is the backbone of modern software delivery.

Once applications are built, they must be tested, integrated, and deployed automatically.  
This is where CI/CD tools come into the picture.

## This module focuses on:

- Understanding how CI/CD pipelines work in real environments  
- Building pipelines using GitHub Actions, GitLab CI/CD, and Jenkins  
- Automating build, test, and deployment workflows  
- Working with runners, agents, and distributed builds  
- Securing pipelines using secrets, RBAC, and Vault  
- Triggering pipelines using webhooks  
- Designing production-grade CI/CD architectures  
- Troubleshooting pipeline failures and real-world issues  

This is a hands-on, implementation-focused module aligned with real industry practices.

## What You Will Learn

- How CI/CD fits into the DevOps lifecycle  
- Pipeline structure (Build → Test → Deploy → Notify)  
- GitHub Actions workflow design and execution  
- GitLab CI/CD pipelines, runners, caching, and artifacts  
- Jenkins architecture (master-agent setup)  
- Freestyle jobs vs pipeline jobs in Jenkins  
- Declarative pipelines and Jenkinsfile usage  
- Multibranch pipelines for automated workflows  
- Webhook-based automation vs polling  
- Using Docker as Jenkins agents  
- Parameterized builds and environment variables  
- Managing secrets securely in pipelines  
- Role-Based Access Control (RBAC) in Jenkins  
- Integrating external secret managers like HashiCorp Vault  
- Setting up secure CI/CD with TLS and domain configuration  
- Reusable pipeline design using Jenkins shared libraries  
- Debugging failed pipelines and handling production issues  

## How to Complete This Module

- Complete all tasks listed in tasks.md  
- Use real application repositories for pipeline implementation  
- Build and test applications through pipelines  
- Automate deployments to a VM or cloud instance  
- Configure webhooks to trigger pipelines automatically  
- Use environment variables and secrets securely  
- Capture logs, pipeline outputs, and failures  
- Document configurations and troubleshooting steps  
- Push all pipeline code and configs to your repository  
- Share your learning publicly (mandatory)  

Pipelines will fail during this module.  
Understanding why they fail and fixing them is a core DevOps skill.

## Outcome of This Module

After completing this module, you will be able to:

- Design and implement CI/CD pipelines across multiple tools  
- Automate application build, test, and deployment workflows  
- Work with GitHub Actions, GitLab CI/CD, and Jenkins confidently  
- Configure runners, agents, and distributed build systems  
- Secure pipelines using RBAC, secrets, and Vault  
- Trigger pipelines using webhooks and events  
- Debug and fix real-world CI/CD failures  
- Build production-ready CI/CD workflows
