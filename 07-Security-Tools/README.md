# Module 7 – Security Tools (Trivy, OWASP, Prowler, SBOM)

Security is a core responsibility in modern DevOps.

Applications are no longer just built and deployed — they must be secure at every stage of the lifecycle. This includes scanning code, dependencies, containers, and cloud infrastructure.

## This module focuses on:

- Understanding security in DevOps (DevSecOps)  
- Identifying vulnerabilities in applications and containers  
- Scanning infrastructure for misconfigurations  
- Using industry tools for real-world security checks  
- Generating and analyzing SBOM (Software Bill of Materials)  
- Integrating security into CI/CD pipelines  

This is a hands-on, security-focused module aligned with real industry practices.

## What You Will Learn

- What DevSecOps is and why it matters  
- Common vulnerabilities and misconfigurations  
- Using Trivy for container and filesystem scanning  
- OWASP dependency and vulnerability checks  
- Cloud security scanning using Prowler  
- Understanding CVEs and vulnerability severity levels  
- Fixing vulnerabilities in applications and images  
- Generating and using SBOM reports  
- Integrating security tools into CI/CD pipelines  
- Security best practices for production systems  

## How to Complete This Module

- Complete all tasks listed in tasks.md  
- Install and run security tools locally or on a VM  
- Scan applications, containers, and systems  
- Identify vulnerabilities and fix them  
- Generate and analyze security reports  
- Integrate scanning into CI/CD pipelines  
- Document findings and improvements  
- Push your work to your repository  
- Share your learning publicly  

Security issues will appear frequently.  
Understanding them and fixing them is a critical DevOps skill.

## Outcome of This Module

After completing this module, you will be able to:

- Scan applications and containers for vulnerabilities  
- Identify and fix security issues  
- Use tools like Trivy, OWASP checks, and Prowler  
- Generate and analyze SBOM reports  
- Integrate security into CI/CD pipelines  
- Apply DevSecOps practices in real projects  
- Build more secure and production-ready systems
